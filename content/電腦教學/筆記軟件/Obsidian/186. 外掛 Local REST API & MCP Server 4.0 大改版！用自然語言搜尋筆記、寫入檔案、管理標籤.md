### 1.1. `Obsidian CLI` 間的差異

問題: `MCP` 工具和 `Obsidian CLI` 有何差異?
`MCP` 工具更直接,`Obsidian CLI` 透過 `Skill` 運作
以下為 Obsidian CLI 與 Local REST API & MCP Server 的功能與規格對比表格：

| 面向 | Obsidian CLI (obsidian 命令) | Local REST API & MCP Server |
| :--- | :--- | :--- |
| **性質** | Obsidian 官方內建功能 | 第三方社群插件 (`coddingtonbear/obsidian-local-rest-api`) |
| **啟用方式** | 設定 &rarr; 一般 &rarr; 啟用 Command line interface | 安裝社群插件 &rarr; 設定 &rarr; Local REST API |
| **通訊協定** | 專有 CLI 協議（子行程通訊） | HTTP(S) REST API + MCP (Streamable HTTP) |
| **端點** | 無（直接與 Obsidian 程序通訊） | `http://127.0.0.1:27123/` (HTTP) /<br>`https://127.0.0.1:27124/` (HTTPS) |
| **認證** | 無（OS 層級行程通訊） | Bearer token (API key) |
| **特點** | 有開發者工具、插件管理、主題、Sync、Publish 等功能 | `obsidian_vault_patch` 可以精準定位 heading/block/frontmatter 做增刪改 |
"""

## 2. MCP Server 功能與設定

### 2.1. MCP 提供功能

| 工具 | 功能 |
| :--- | :--- |
| obsidian_vault_list | 列出資料夾內容 |
| obsidian_vault_read | 讀取檔案內容 |
| obsidian_vault_write | 寫入檔案 |
| obsidian_vault_patch | 修補檔案特定區段 |
| obsidian_vault_delete | 刪除檔案 |
| obsidian_vault_move | 移動/重新命名檔案 |
| obsidian_vault_get_document_map | 取得文件結構（標題、區塊參考、frontmatter） |
| obsidian_vault_append | 附加內容到檔案結尾 |
| obsidian_search_simple | 簡易搜尋 |
| obsidian_search_query | JsonLogic 進階搜尋 |
| obsidian_tag_list | 列出所有標籤 |

### 2.2. AI Agent 設定

詳細設定與說明可以到 [RESET API GITHUB](https://github.com/coddingtonbear/obsidian-local-rest-api) 參考。

- `Cursor` 設定如下:

```
{
  "mcpServers": {
    "obsidian": {
      "url": "https://127.0.0.1:27124/mcp/",
      "headers": {
        "Authorization": "Bearer <your-api-key>"
      }
    }
  }
}
```

- `Claude` 設定如下:

  ```yaml
  {
    "mcpServers": {
      "obsidian": {
        "type": "http",
        "url": "https://127.0.0.1:27124/mcp/",
        "headers": {
          "Authorization": "Bearer aa70a05e7206b1b0e45462c92d878ae4a098e4cd3574f4745dc881e62d85985e"
        }
      }
    }
  }
  ```

- `OpenCode` 設定如下: 
  - `Linux/Mac`: `~/.config/opencode/opencode.json`
  - `Windows`: `c:\Users\帳號\.config\opencode\opencode.json`

    ```json

	"mcp": [

		"obsidian": (

			"type": "remote",

			"url": "http://127.0.0.1:27123/mcp/",

			"enabled": true,

			"headers": {

				"Authorization": "Bearer YOUR_API_KEY",
           }
    }      
	```

 - 因爲 `Local REST API` 使用[自簽憑證](https://127.0.0.1:27124/obsidian-local-rest-api.crt) ,直接使用 **http://** 比較簡單
 - 啟動 `opencode` 前,先執行下列命令的話, `"url"` 即可使用 "https://127.0.0.1:27124/mcp/"`
   
   ```yaml
   set NODE_TLS_REJECT_UNAUTHORIZED=0
   
   REM 永久設定環境變數
   setx NODE_TLS_REJECT_UNAUTHORIZED 0
   ```

### 4.2.Local REST API 4.x 新設定

- 新增内容: PATCH/vault/資料夾/筆記.md
  - 在筆記裡找到 `2026` 標題,新增內容到標題內的開頭

  **HTTP Headers** 欄位內輸入:

  ```yaml
  Operation: prepend
  Target-Type: heading
  Target: Misc::2026
  Content-Type: text/markdown
  ```

- Operation:append(加在標題內最後面),prepend(加在標題内第一行),replace
- Target-Type: heading, block, frontmatter
- heading 時必須用 heading1: :heading2 的格式,由上到下指定heading
- Content-Type: 不能使用 `text/plain` 一可使用 `text/markdown` 或 `application/json`
