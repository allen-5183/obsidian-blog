## 安裝 `SQL Server`

## 下載 `SQL SERVER`

1. **至官網下載安裝軟體**
   到以下連結下載 `SQL SERVER` 安裝程式
   `https://www.microsoft.com/zh-tw/sql-server/sql-server-downloads`

   或是在 `Google` 中搜尋 `SQL Server`
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_1.png#w60)

2. **找到 `開發人員版本/Standard` 版，並按下下載**
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_2.png#w60)

3. **執行下載的安裝程式**
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_3.png#w60)

4. **選擇安裝類型**
   選擇安裝類型為 **Down Media**
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_4.png#w60)

5. **選擇下載選項**
   - `SELECT LANGUAGE(選取語言)`: **繁體中文**
   - `WHICH PACKAGE WOULD YOU LIKE TO DOWNLOAD(下載哪類型的套件)`: **ISO**
   - `SELECT DOWNLOAD LOCATION(下載位置)`: **C:\Users\User\Downloads**
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_5.png#w60)

## 安裝 `SQL SERVER`

1. **將 `SQLServer iso` 掛載成光碟機**
   執行掛接或使用開檔案總管開放下載的 `iso` 檔案,搭成光碟機
   ![](https://pic.allen5183.synology.me/SQL_DOWNLOAD_6.png#w60)
   >若無**掛接**選項可操作，代表這個 .iso 檔不是為檔案總管類型開啟，而是被其他應用程式，解決方式:右鍵 .iso →「開啟方式」 →「選擇其他應用程式」選：`Windows` 檔案總管，並勾： **一律使用此應用程式開啟 .iso**

2. **執行 `setup.exe` 安裝 `SQL Server`**
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_7.png#w60)

3. **選擇安裝新伺服器**
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_8.png#w60)

4. **選擇安裝版本**
   若無安裝序號可選擇免費版本的 `Developer` (開發版)，有序號可直接輸入安裝序號。
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_9.png#w60)
   >如果無法執行，可能是 ` Windows Update` 被關閉了，可以到以下網址下載管理工具 `https://www.sordum.org/files/downloads.php?st-windows-update-blocker`，執行 `wub_x64.exe` 程式，並開啟 `windows Update` 功能

5. **同意授權條款**
   勾選 同意 授權條款
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_10.png#w60)

6. **安裝規則**
   這裡若有開啟防火牆時，會出現警告畫面，直接按下一步，即可，否則會略果

7. **`Azure` 延伸模組**
   移除 `Azure` 延伸模組相關的選項
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_11.png#w60)

8. **特徵選取**
   按下全選並移除機器學習及外部 `Polybase` 及 `SSAS` 相關的選項
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_12.png#w60)

9. **執行個體 `Instant` 組態**
   `SQL Server` 執行個體使用**預設**名稱與**具名**之差異

   特徵(`Feature`)|預設執行個體(`Default Instance`)|具名(`Named Instance`)
   --|--|--
   每台伺服器數量|只能有一個|可以有多個
   連線字串|`ServerName`|`ServerName\InstanceName`
   網路連接埠|預設 `TCP 1433`|預設為動態埠
   `SQL Server Browser` 服務|不需要|需要
   最佳使用情境|專用資料庫伺服器、開發環境|在單一主機上整合多個資料庫、多版本共存

   這裡選擇具名執行個體: `SQL2025`
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_13.png#w60)

   >注意事項:
    預設執行個體
    使用 **電腦名稱(或IP)** 即可連到此執行個體，但只能給一個執行個體使用
    例如： `**dbserver**`<br>
    具名執行個體
    輸入 **電腦名稱(或IP)** 後需要再輸入一個　**識別的代號**　才能連到此執行個體
    例如： `dbserver/SQL2025`

10. **伺服器組態**
    設定以下三項為 自動，其餘皆 手動
    - `SQL Server Agent`
    - `SQL Server Database Engine`
    - `SQL Server Browser`
   ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_14.png#w60)

11. **資料庫引擎組態**
    - 驗證模式
     「驗證模式」決定了「誰」以及「如何」可以登入您的資料庫伺服器。一個正確的選擇可以簡化管理、提升安全性；反之，則可能為日後的維運埋下不必要的風險。
      ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_15.png#w60)

      >注意事項:
       `Windows` 驗證模式：
       只能使用 `Windows` 的 使用者 登入 `SQL Server`<br>
       `混合模式 (SQL Server 驗證與 Windows 驗證)`：
       可以使用 `Windows` 的 使用者 或 `SQL Server` 的 使用者 登入 `SQL Server`
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_16.png#w60)

       加入以下兩個帳號
       - 目前使用者
       - `administrator` 帳號當作是備援的管理帳號
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_17.png#w60)
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_18.png#w60)
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_19.png#w60)
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_20.png#w60)
       ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_21.png#w60)

       >`SQL Server` 管理員為 `sa` 的備援帳號，擁有最高的權限，假如 `sa` 的密碼忘記了，還可以用 `SQL Server` 管理員來登入並修改

       >啟用 administrator 帳號
        `net user administrator /active:yes`

12. **`Integration Service Scale Out` 組態**
    `Integration Service Scale Out` 組態設定不變
    ![1](https://pic.allen5183.synology.me/SQL_DOWNLOAD_22.png#w60)
